# RMG ammonia–methane oxidation mechanism — round-trip test fixture

These three files are a real, unmodified [RMG](https://reactionmechanismgenerator.github.io/)
(Reaction Mechanism Generator) output for an **ammonia–methane (NH₃/CH₄)
oxidation** system, used as the real-data fixture for the CHEMKIN
interoperability round-trip test
(`tests/integration/test_chemkin_round_trip_real.py`).

| File | Role |
|---|---|
| `chem.inp` | CHEMKIN mechanism: `ELEMENTS` / `SPECIES` / `THERM ALL` (inline NASA-7) / `REACTIONS` (KCAL/MOLE, MOLES) |
| `species_dictionary.txt` | RMG adjacency-list dictionary — the structure map used to resolve each species name to canonical SMILES + multiplicity |
| `tran.dat` | Lennard-Jones transport (σ, ε/kB, dipole, polarizability, rotational relaxation) |

**Content:** 21 species (NH₃, CH₄, O₂, NH₂, NO, CN, HO₂, CH₃, CH₂O, H₂O₂,
C₂H₆, …), 64 reactions exercising real modified-Arrhenius, Troe falloff,
Chebyshev pressure-dependence (written by RMG as `R(+M)<=>P(+M)` +
`TCHEB/PCHEB/CHEB`), duplicate reactions, and transport.

**Provenance:** internal TCKDB RMG run (working label `OriginalEnas_L1`),
generated by a student to emulate the mechanism/experiments of:

> P. García-Ruiz, I. Salas, E. Casanova, R. Bilbao, M. U. Alzueta,
> *Experimental and Modeling High-Pressure Study of Ammonia–Methane
> Oxidation in a Flow Reactor*, **Energy & Fuels** 38 (2024) 1399–1415.

The RMG mechanism is a generated emulation, not the published mechanism
itself; it is committed here only as an interoperability test fixture. It is
included in-repo (rather than referenced externally) so CI is reproducible,
following the repo convention of internalizing external data that becomes
part of the test suite.

**Why real data:** hand-authored CHEMKIN fixtures test the format's happy
path; a real RMG file exercises the dialect corners production tools actually
emit. Wiring this fixture in surfaced five importer/exporter bugs that toy
fixtures could not — the `THERM ALL` keyword abbreviation, `(+M)`+Chebyshev
notation, the dummy `1.0 0 0` main line on Chebyshev rows, Chebyshev needing
`(+M)` on export, and the missing `DUPLICATE` keyword on export.
